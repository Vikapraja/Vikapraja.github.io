*form's Attributes:
  • name="..."

    =>In form name attribute is used to get value of input in backend programming languages like java,python,php etc. =>Name Attribute's value must be unique in each element.
    =>name attribute can be used only in input related element like 'form', 'input-selector', 'textarea'.
    =>This attribute will not use in non-input type elements like headings, paragraph,list,table etc.

  • id="..."

    =>id="" attribute is used to select elements in css and js(javascdript) to apply stylesheets scripts respectvily.
    =>This attribute's value must be unique in each elements.
    =>We can use Id attribute in both input & non-input elements.

  • class="..."

    =>class attribute is used to select multiple elements to apply css and stylesheet's scripts.
    =>class attribute is used to apply common stylesheet and css scripts on multiple elements
    =>we can use single class in multiple elements As well as we can use multiple class in a single elements.

  • action=""

    =>It is for backend purpose.
    =>Action attribute is used to provide url of particular web-page/backend ApI where we want submit form's data.
    =>Keep it blank to redirect same url in send data, on same-page.

  • method="GET/POST"

    =>method attribute has two value -> GET/POST.
    GET: > corresponds to the htttp by GET method .
    =>form data are appende to the acction attribute's URL with a '?' as separator ,and the resulting URL is select to server.
    =>use this method when the form has no side-effects contains only ASCII chars.
    POST:> corresponds to the htttps POST method.
    =>Form's data are included in the body of the form and send to server. .dailog:> dailog is used when the form is inside a dailog element.

  • enctype="(application/x-www-form-urlencoded)or(multipart/form-data)or(multipart/form-data)"

    =>When the value of the method attribute is POST, enctype is the MIME type of content that is used to submit the forms to the server.
    =>'application/x-www-form-urlencoded' >The default value if the attribute is not specified.
    =>'multipart/form-data' > the value used for an 'input' element with the type attribute set to 'file'.
    =>"text/plain" > (html5)
    =>The value can be overriden by a formenctype attribute on a 'button' or 'input' element.

4)button
*button's Attributes
  • type="submit/reset/button"

    • =>button's attribute {type="" } values = submit/reset/button.
    =>type's value submit :>The button type 'submit' submit the form's data to server. This is the default if the attribute is not specified or if the attribute is dynamically change to an empty or invailid value.
    =>reset :> Clicking a reset button will clear all input elements in the form to their orginal value.The button can contain text/images/icons/and other elements.
    =>type's value button :> It defines a clickable button only. Mostly used with a JavaScript to activate a script.
