#OOPs Concept in JavaScript (Object Oriented Programming Structure)

OOPs is Designed by Bjane Stroustroup (C++ founder) =>Object Oriented Programming Structure (OOPS) is a computer science term which defines a well structured serise or steps to construct a Computer Application which composed of multiple objects, each object can handle data, get and transfer message to each other, since object are not dependent on each other oops is seen as being more flexible the procedural approach.

  • =>OOPS Approachs :
    1. Class & Objects
    2. Abstraction [Data Hidding]
    3. Encapsulation [Binding data and function in a single unit]
    4. Polymorephism [Many Forms of single Entity]
    5. Inheritance [Code Reusebility]
  1. Class :
    • =>class is required to implement opps.
    • =>Class is technique by which we can define new data-type as we required.
    • =>Class is blue print of Object.
    • =>Class is collection of property and behavior of object where properties means variable of different-different data-types and behavior means various functions.
    • =>Class is a logical entity only that means class doesn't have physical existance in memory, until its instance is created and name called object.

  3. Abstraction :
    • =>Data Hiding i.e hiding complexity showing functionality.
    • =>We can hide members using access specifiers like public,private and protected,
    • =>(javaScript Doesn't Support Abstraction)
  4. Encapsulation
    • =>Binding data members with Member function/methods OR
    • =>Wrapping-up data members with member function is called Encapsulation.
    • =>Binding code and data togather into a single unit are known as Encapsulation.
  5. Polymorephism
    • =>Many forms of single entity.
    • =>If one task is performed in different ways, it is known asPolymorephism.
    • =>Exmple of Polymorephism :
      1. Method Overloading
      2. Constructor & Destructor
      3. Method Overridding
      4. Operator Overloading
      • Method Overloading > if we define more than one function in a class with same name but with different signature (either number of arguments must be different or their types must be different) (JavaScript Doesn't Support Method Overloading)
      • Constructor & Destructor > we can use constructor in JavaScript but we can't use destructor in JS
      • Method Overriding > if both parent class and child class having same named members then child class override parent's member i.e child class can inherit only those member from parent which child doesn't have JS Support This Concept
      • Operator Overloading > if we provide user defined defination to an existing operator then this concept is called operator overloading JS Doesn't Support this feature

  6. Inheritance
    • =>Inheritance allows us to define class that inherits all methods and properties from another class.
    • => extends keyword are used.
    • =>if we reuse member of any existing class in a new class is called Inheritance.
    • =>Parent Class is that class which is being inherit , also called base class
    • =>Child Class is that class which inherit to anoother class, also called derrived class.
    • => 
      Inheritance Syntax :
                                            class class1{
                                                .........
                                            }
                                            class class2 extends class1{
                                                .........
                                            }
    • =>Types of Inheritance:
      1. Single Inheritance
      2. Multi-Level Inheritance
